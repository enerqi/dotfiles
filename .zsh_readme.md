## Zsh Configuration in This Dotfiles

When Zsh starts, it reads several configuration files in a specific order. Understanding this order helps keep your dotfiles clean, maintainable, and predictable.

### Loading Order

The configuration files are read in the following order:

1. **`/etc/zshenv`**
2. **`${ZDOTDIR:-$HOME}/.zshenv`**
3. **`/etc/zprofile`**
4. **`${ZDOTDIR:-$HOME}/.zprofile`**
5. **`/etc/zshrc`**
6. **`${ZDOTDIR:-$HOME}/.zshrc`**
7. **`/etc/zlogin`**
8. **`${ZDOTDIR:-$HOME}/.zlogin`**
9. **`${ZDOTDIR:-$HOME}/.zlogout`**
10. **`/etc/zlogout`**

### Summary Table

| File                    | Sourced For                  | When It Runs                  | Recommended Content                                      | Used in This Dotfiles |
|-------------------------|------------------------------|-------------------------------|----------------------------------------------------------|-----------------------|
| `.zshenv`               | All shells                   | Always (even scripts)         | Environment variables only (`PATH`, `EDITOR`, etc.)     | Yes (keep minimal)    |
| `.zprofile`             | Login shells                 | Before `.zshrc`               | Login-time environment, ssh-agent, PATH tweaks           | Yes (light)           |
| `.zshrc`                | Interactive shells           | Main interactive config       | Aliases, functions, plugins, prompt, key bindings, etc. | **Main file**         |
| `.zlogin`               | Login shells                 | After `.zshrc`                | One-time login messages (`fortune`, etc.)                | Minimal / optional    |
| `.zlogout`              | Login shells                 | On logout                     | Cleanup tasks                                            | Rarely used           |

### Detailed Description of Core Zsh Files

#### `.zshenv`
This file is sourced by **all** instances of Zsh, including non-interactive scripts.
It should be kept as small as possible and should **only** define environment variables.

#### `.zprofile`
Sourced for **login shells** before `.zshrc`.
Originally added for KornShell compatibility.

#### `.zshrc`
The main configuration file for **interactive shells**.
This is where most customization lives (plugins, aliases, key bindings, completions, etc.).

#### `.zlogin` / `.zlogout`
`.zlogin` runs after `.zshrc` for login shells (one-time login actions).
`.zlogout` runs on logout for cleanup.

### Key Tools & Frameworks in This Setup

This dotfiles uses a modern, performance-oriented stack. Here's what each major component does and why it's included:

#### **zgenom** â€“ Plugin Manager
- **Role**: Lightweight and fast Zsh plugin manager (a maintained fork/superset of the original `zgen`).
- **How it works**: On first run (or after `zgenom reset`), it clones plugins and generates a **static init script**. On subsequent shells, it simply sources this pre-compiled script instead of re-checking repositories or running slow logic every time.
- **Benefits**: Dramatically faster startup compared to plugin managers that do work on every shell launch. Supports Oh My Zsh and Prezto natively, includes autoupdate (in background), `zgenom compile`, and `zgenom clean`.
- **In this setup**: It manages the entire plugin ecosystem and is the foundation for loading Prezto and other tools.

#### **Prezto** â€“ Zsh Configuration Framework
- **Role**: A modular configuration framework that provides sane defaults, useful aliases, functions, improved history/directory handling, and a solid base for the completion system.
- **Philosophy**: "Instantly awesome" Zsh without the bloat of Oh My Zsh. You load only the modules you need (e.g., `environment`, `editor`, `history`, `directory`, `utility`, `completion`, `git`, `syntax-highlighting`, `autosuggestions`).
- **Benefits**: Cleaner and generally faster than Oh My Zsh. Provides the core `compsys` (completion system) that tools like **carapace** and **fzf-tab** build upon.
- **In this setup**: Prezto forms the foundational layer. Many modules are loaded via zgenom; the prompt module is disabled because Starship replaces it.

#### **Oh My Zsh** (Now Mostly Redundant)
- **Role** (historically): Popular framework that provided plugins, themes, and helpers.
- **Why it's largely redundant now**: Prezto provides better modular defaults, while **carapace** (completions), **starship** (prompt), **fzf-tab**, and **zoxide** offer superior, faster alternatives to OMZ's equivalents. The `mise` plugin was also made lazy to avoid slowdowns.

#### **zoxide** â€“ Smarter `cd` Command
- **Role**: A fast, Rust-based alternative to `z` / `autojump`. It tracks your most frequently visited directories and lets you jump to them with fuzzy partial matching.
- **Usage examples**:
  - `z dotfiles` â†’ jumps to your most-used "dotfiles" directory
  - `zi conf` â†’ interactive selection with fzf
- **In this setup**: Initialized with `eval "$(zoxide init zsh)"` near the end of `.zshrc`. It greatly improves daily navigation speed.

#### **Starship** â€“ Cross-Shell Prompt
- **Role**: A minimal, extremely fast, and highly customizable prompt written in Rust.
- **Benefits**:
  - Works identically across Zsh, Bash, Fish, etc.
  - Very low performance overhead (much faster than traditional Zsh themes or Oh My Zsh prompts).
  - Configured via a single `~/.config/starship.toml` file with rich modules (git status, language versions, Docker, Kubernetes, etc.).
- **In this setup**: The Prezto prompt module is disabled, and `eval "$(starship init zsh)"` is used instead. Requires a Nerd Font for best results.

#### **Carapace** â€“ Modern Completion Engine
- **Role**: A fast, multi-shell completion binary that provides high-quality, context-aware completions for hundreds of CLI tools (git, kubectl, docker, cargo, etc.).
- **Benefits**:
  - Much faster than traditional Zsh script-based completions (`zsh-completions`).
  - Consistent completions across different shells.
  - Bridges other completion systems via `CARAPACE_BRIDGES`.
  - Works beautifully with **fzf-tab** for a fuzzy, searchable completion menu.
- **In this setup**: Loaded after a lightweight `compinit`, replacing the heavy `zsh-users/zsh-completions` bundle. This is one of the biggest contributors to improved shell startup time.

### Best Practices in This Dotfiles

- Keep `.zshenv` and `.zprofile` **light** â€” only environment variables and login-specific setup.
- Put all interactive configuration in **`.zshrc`**.
- Use **`.zshrc.local`** (and `.zprofile.local`) for machine-specific or private settings.
- After changing plugins, run `zgenom reset` followed by `exec zsh`.
- Monitor performance with `zmodload zsh/zprof` at the top and `zprof` at the bottom of `.zshrc`.

This combination (zgenom + Prezto + carapace + starship + zoxide + fzf-tab) gives you a fast, modern, and highly productive Zsh experience with minimal bloat.
